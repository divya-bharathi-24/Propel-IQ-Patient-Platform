## Category: Actions

### C/Actions/Button

**Purpose**: Primary interactive trigger for user actions.

#### Variants (Type × Size × State)

| Variant | Description |
|---------|-------------|
| Primary | High emphasis — one per view maximum |
| Secondary | Medium emphasis — outline style |
| Ghost | Low emphasis — transparent background |
| Destructive | Dangerous/irreversible actions |

| Size | Height | Padding | Font Token |
|------|--------|---------|-----------|
| Small (S) | 32px | 8px 12px | `type.label.sm` |
| Medium (M) | 40px | 10px 16px | `type.label.md` |
| Large (L) | 48px | 12px 24px | `type.body.sm` (600) |

| State | Visual |
|-------|--------|
| Default | Base styling |
| Hover | Background darkens / border color shift (150ms) |
| Focus | 2px primary focus ring, 2px offset |
| Active | Pressed / scale 0.98 |
| Disabled | 40% opacity, `cursor: not-allowed` |
| Loading | Spinner replaces label text; dimensions preserved |

**Props**:
- `label`: string (required)
- `type`: Primary | Secondary | Ghost | Destructive
- `size`: S | M | L
- `icon`: None | Leading | Trailing
- `iconName`: Lucide icon name
- `loading`: boolean
- `disabled`: boolean
- `fullWidth`: boolean

**Usage Rules**:
- Only one Primary button per view/section
- Destructive buttons must be at least one click away from a confirmation dialog
- Loading state disables the button and shows spinner; never remove button from DOM

**Angular Path**: `app/src/app/shared/components/button/button.component.ts`
**Figma Naming**: `C/Actions/Button/[Primary|Secondary|Ghost|Destructive]/[S|M|L]/[Default|Hover|Focus|Active|Disabled|Loading]`

**Purpose**: Compact action button with icon only (no label).

## Category: Inputs

### C/Inputs/TextField

**Purpose**: Single-line text entry for forms.

| Size | Height | Font |
|------|--------|------|
| Small | 32px | `type.body.sm` |
| Medium | 40px | `type.body.md` |
| Large | 48px | `type.body.md` |

**Anatomy**:
1. Floating label (above field on focus/fill)
2. Input area with placeholder
3. Leading/trailing icon (optional)
4. Helper text (below field)
5. Error message (replaces helper text on error)
6. Character counter (optional)

**States**: Default, Focus, Filled, Error, Disabled, Masked (PHI)

**Masked (PHI) variant**:
- Value shows as `•••••••••`
- Trailing EyeOff icon button to reveal
- `aria-label` includes "masked, click eye icon to reveal"

**Props**: `label`, `placeholder`, `value`, `helperText`, `errorMessage`, `masked`, `leadingIcon`, `trailingIcon`, `maxLength`, `required`, `disabled`, `size`

**Angular Path**: `app/src/app/shared/components/text-field/text-field.component.ts`

---

### C/Inputs/TextArea

**Purpose**: Multi-line text input for clinical notes, long-form responses.

**Min height**: 80px (3 rows)
**Resize**: Vertical only
**Max rows**: 12 (auto-grows, then scrolls)
**States**: Default, Focus, Error, Disabled

---

### C/Inputs/Select

**Purpose**: Single-option dropdown selector.

**Height**: 40px (medium)
**Dropdown**: Opens below, max height 240px, scrollable
**Search**: Searchable variant for long lists (>20 options)
**States**: Default, Open, Focus, Error, Disabled

**Usage**: Specialty selection, role assignment, filter dropdowns.

---

### C/Inputs/Checkbox

**Size**: 18×18px checkbox + label
**States**: Unchecked, Checked, Indeterminate, Disabled
**Group**: Checkbox group with legend for accessibility

**Usage**: Consent agreements on registration, batch operations in user management.

---

### C/Inputs/Radio

**Size**: 18×18px radio button + label
**States**: Unselected, Selected, Disabled
**Group**: Radio group with `fieldset`/`legend` for accessibility

**Usage**: Intake mode selection (AI / Manual), calendar service selection.

---

### C/Inputs/Toggle

**Purpose**: Binary on/off switch.
**Size**: 44×24px track, 20×20px thumb
**States**: Off/Default, On/Active, Disabled
**Label**: Always has visible text label; not icon-only

**Usage**: Intake mode toggle (AI ↔ Manual), notification preferences.

---

### C/Inputs/DatePicker

**Purpose**: Date selection with calendar popover.

**Variants**:
- Single date (appointment booking)
- Date range (audit log filters)
- Month/year picker (report filters)

**Calendar grid**: 7-column week grid, current day highlighted, unavailable dates grayed
**Keyboard**: Full keyboard navigation (arrow keys, Enter to select)
**Mobile**: Opens as bottom sheet on ≤768px

---

### C/Inputs/SearchBar

**Purpose**: Real-time search input for tables and lists.

**Height**: 40px
**Leading icon**: Search (magnifying glass)
**Clear button**: Trailing × icon when value present
**Debounce**: 300ms before triggering search

**Usage**: Patient search in walk-in booking, user search in admin, audit log filter.

---

### C/Inputs/DropZone (PDF Upload)

**Purpose**: Drag-and-drop file upload zone for clinical documents.

**Anatomy**:
1. Dashed border zone (400×160px default, flexible)
2. Upload icon (Upload, 32px)
3. Primary text: "Drag & drop PDF files here"
4. Secondary text: "or click to browse — Max 25MB per file, up to 20 files"
5. File list (below zone) showing upload progress per file

**States**:
- Default: Dashed border, `color.border.default`
- Drag over: Solid border `color.primary`, `color.primary.surface` background
- Uploading: Progress bars per file
- Success: Green checkmark per file
- Error (per file): Red × icon + error message; successful files unaffected

**Keyboard**: Activates file picker on Enter/Space; `aria-label="Upload PDF files"`

**Props**: `maxFileSizeMB`, `maxFiles`, `acceptedTypes`, `onFilesSelected`

## Category: Navigation

### C/Navigation/Header

**Purpose**: Top application bar.

**Height**: 64px (desktop) / 56px (mobile)
**Contents**:
- Left: Logo / app name
- Center: Page title (desktop) / empty (mobile — title in sidebar)

**Purpose**: Primary navigation panel for authenticated users.

**Width**: 240px (expanded) / 64px (collapsed, icon only)
**Animation**: Slide (300ms ease-out)

**Navigation items by role**:

| Patient | Staff | Admin |
|---------|-------|-------|
| Dashboard | Dashboard | User Management |
| Appointments | Same-Day Queue | Audit Log |
| Intake | Walk-In Booking | — |
| Documents | Patient Records | — |
| Profile | — | — |

**Item states**: Default, Hover, Active (current page), Disabled
**Active indicator**: Left border 3px `color.primary` + `color.primary.surface` background

---

### C/Navigation/BottomNav (Mobile)

**Purpose**: Mobile navigation bar replacing sidebar at ≤768px.

**Height**: 56px + safe area bottom
**Max items**: 5 tabs
**States**: Default, Active
**Active indicator**: Icon color `color.primary` + label shown

**Patient tabs**: Home, Appointments, Intake, Documents, Profile
**Staff tabs**: Dashboard, Queue, Walk-In, Records
**Admin tabs**: Users, Audit Log

---

### C/Navigation/Tabs

**Purpose**: In-page tabbed content switching.

**Height**: 48px tab bar
**Underline**: 2px `color.primary` on active tab
**Animation**: 200ms slide on indicator
**Overflow**: Horizontal scroll on mobile

**Usage**:
- Patient 360-degree view: Vitals | Medications | Diagnoses | Allergies | History
- Medical code review: ICD-10 | CPT
- Admin user management: Active | Inactive

---

### C/Navigation/Breadcrumb

**Purpose**: Hierarchical location indicator.

**Separator**: `/` or `>` chevron icon
**Max depth shown**: 3 levels (root → section → current)
**Current page**: Non-linked, `color.text.primary`
**Parent pages**: Linked, `color.text.secondary`
**Mobile**: Hidden (≤768px)

## Category: Content

### C/Content/Card

**Base spec**: See Design System Section 8.5

**Variants**:

#### Appointment Card (Patient-facing)
- Date + time (H5)
- Provider specialty
- Status badge
- Actions: "Cancel" (Ghost/Destructive), "Reschedule" (Ghost), "Add to Calendar" (Ghost)

#### Stats Card (Dashboard)
- Icon (24px, colored)
- Metric value (H2)
- Label (`type.body.sm`)
- Trend arrow + % change

#### Clinical Data Card (360-degree view)
- Category icon
- Field name (`type.label.md`)
- Field value (`type.body.md`)
- Source trace tag
- Conflict badge (if applicable)
- AI confidence badge (if AI-extracted)

#### Patient Card (Staff view)
- Avatar (40px)
- Name (`type.h5`)
- DOB (masked by default)
- Risk badge
- Quick actions

#### Booking Confirmation Card
- Booking reference number (monospace)
- Date, time, provider
- "Download PDF" link
- "Add to Calendar" button

---

### C/Content/Table

**Purpose**: Tabular data display for lists and management views.

**Structure**: Header row + data rows + optional footer (pagination)

| Property | Value |
|----------|-------|
| Header | `color.bg.subtle`, `type.label.md`, sticky on scroll |
| Row height | 48px |
| Hover | `color.bg.subtle` |
| Selected row | `color.primary.surface` |
| Sort icon | ChevronUp/Down, 16px, `color.text.secondary` |

**Responsive**: Transforms to card-list at ≤768px (all columns as `label: value` pairs).

**Variants**:
- Queue Table (SCR-014): Name, Time, Type, Risk, Status, Actions
- Code Review Table (SCR-017): Code, Description, Evidence, AI Confidence, Staff Action
- Audit Log Table (SCR-021): Timestamp, User, Role, Action, Patient Record
- User Management Table (SCR-020): Name, Email, Role, Status, Last Login, Actions

**States**: Default, Hover (for interactive avatars)

## Category: Feedback

### C/Feedback/Modal

**Purpose**: Focused overlay for critical interactions.

See Design System Section 8.7.

**Required anatomy**:
1. Overlay (50% black)
2. Modal panel (centered)
3. Header: title + close (×) button
4. Body: content
5. Footer: action buttons (right-aligned)

**Variants**:
- Default (480px): Confirmation, forms
- Large (640px): Complex forms, code review
- Session Timeout (480px): Countdown timer + actions

**Accessibility**: `role="dialog"`, `aria-modal="true"`, focus trapped inside, ESC closes.

**Purpose**: Slide-in side panel for supplementary content.

**Width**: 400px (desktop) / 100% (mobile)
**Side**: Right
**Animation**: Slide from right (300ms ease-out)
**Overlay**: Semi-transparent backdrop

**Usage**: Conflict resolution (OVL-005), appointment detail (mobile), filters panel.

### C/Feedback/Skeleton

**Purpose**: Loading placeholder that preserves layout.

**Variants**:
- Text line (width varies, 16px height)
- Paragraph (3–4 text lines)
- Card (matches card dimensions)
- Table row (matches table row height)
- Avatar (circular)
- Slot grid (matches slot cell dimensions)

**Animation**: Shimmer effect (gradient sweeps left to right, 1.5s infinite)
**ARIA**: `aria-busy="true"` on container, `aria-label="Loading content"`

- Determinate (known percentage — file upload)

## Category: Healthcare-Specific

### C/Healthcare/SlotGrid

**Purpose**: Visual appointment slot selector.

**Layout**: Time slots arranged in a grid by day column and time row.
**Cell size**: 80×48px (desktop), 60×40px (tablet)
**Header**: Day labels (Mon, Tue...) + date
**Side labels**: Time labels (8:00 AM, 8:30 AM...)

**States per cell**: Available, Unavailable, Selected, Waitlisted (preferred slot)

**Interaction**:
- Available cell: Click/tap to select (single selection)
- Selected cell: Click again to deselect
- Unavailable cell: Click opens "Add to waitlist" option (preferred slot flow)
- Keyboard: Arrow keys navigate grid; Enter selects

**Props**: `slots: SlotData[]`, `selectedSlotId`, `preferredSlotId`, `onSlotSelect`, `onPreferredSelect`

**Purpose**: No-show risk level indicator for staff views.

**MUST convey meaning through both color AND text AND icon** (never color alone — UXR-103):

| Level | Color | Icon | Label |
|-------|-------|------|-------|
| Low | `color.risk.low` | TrendingDown (16px) | "Low" |
| Medium | `color.risk.medium` | TrendingUp (16px) | "Medium" |
| High | `color.risk.high` | AlertTriangle (16px) + pulse | "High" |

**Props**: `level: 'low' | 'medium' | 'high'`
**ARIA**: `aria-label="No-show risk: [Level]"`

## Usage Rules

1. **Token-only**: All visual properties MUST reference tokens from `designsystem.md`. No raw hex, px, or font values.
2. **State completeness**: Every component MUST implement all defined states before being marked ready for design/dev handoff.
3. **Accessibility first**: Every interactive component MUST have keyboard support, visible focus states, and ARIA labels before shipping.
4. **Healthcare variants**: `C/Healthcare/*` components are domain-specific and MUST NOT be used for generic UI patterns.
5. **PHI protection**: Any component rendering PHI data MUST use the masked variant by default (UXR-402).
6. **No custom components**: Do not create one-off components outside this library. Raise a design review if a new pattern is needed.
7. **Naming discipline**: Follow `C/<Category>/<Name>` Figma naming and `kebab-case.component.ts` Angular naming consistently.
