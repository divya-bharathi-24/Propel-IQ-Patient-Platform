## 10. Accessibility Requirements

### Contrast Ratios (WCAG AA)

| Use Case | Minimum Ratio | Guidance |
|----------|--------------|---------|
| Body text | 4.5:1 | Normal text â‰Ą14px |
| Large text | 3:1 | Bold â‰Ą18px or regular â‰Ą24px |
| UI components | 3:1 | Buttons, inputs, icons |
| Focus indicators | 3:1 | 2px offset focus ring |
| PHI masked fields | 4.5:1 | Masked content must still be legible |

### Focus Management

- All interactive elements have visible focus rings: 2px `color.primary` outline, 2px offset
- Modal/drawer opens: focus moved to modal title or first focusable element
- Modal/drawer closes: focus returns to triggering element
- Form error submission: focus moved to first error field or error summary

## 11. HIPAA Design Patterns

### PHI Masking Pattern

Fields containing Protected Health Information (PHI) MUST follow this pattern:

1. **Default display**: Value shown as `â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘` (masked dots)
2. **Reveal toggle**: Eye icon button (44Ă—44px tap target) next to masked field
3. **Reveal action**: Click/tap to show plain text value; icon changes to EyeOff
4. **Auto-mask**: Field re-masks after 30 seconds of inactivity if unmasked
5. **Audit log**: PHI reveal events logged (user ID, field name, timestamp)

**PHI Fields that MUST use this pattern**:
- Date of birth
- Insurance member ID / group number
- Social Security Number (if collected)
- Phone number (last 7 digits masked by default)

### Audit Trail Visual Indicators

- Audit-logged actions display a small `đź“‹` badge or "Logged" indicator in the UI
- Staff can see audit log indicators on actions they take (viewing records, verifying profiles, confirming codes)

### Data Access Warning

When Staff or Admin accesses a patient's PHI, a discreet banner displays:
> "Accessing protected health information. All activity is logged per HIPAA requirements."

Banner: `color.info.surface`, `type.body.sm`, dismissible per session.
