#### AI Sequence Diagram — Trust-First 360° View Generation

Shows the full RAG-powered aggregation flow for UC-008, including ACL filtering, circuit breaker, and staff verification. Traces to AIR-001, AIR-002, AIR-Q01, AIR-S02, AIR-O02.

```mermaid
sequenceDiagram
    participant Staff as Staff
    participant Frontend as Angular Frontend
    participant Gateway as API Gateway
    participant Clinical as Clinical Module
    participant AIMod as AI Module (Semantic Kernel)
    participant VectorDB as pgvector (PostgreSQL)
    participant OpenAI as OpenAI API (GPT-4o)
    participant Postgres as PostgreSQL

    Note over Staff,Postgres: Trust-First AI — 360° View Generation (AIR-001, AD-6)

    Staff->>Frontend: Open 360° Patient View
    Frontend->>Gateway: GET /api/patients/{id}/360view (JWT)
    Gateway->>Clinical: Route request (RBAC: Staff only)
    Clinical->>Postgres: Fetch ClinicalDocuments + ExtractedData for patient
    Postgres-->>Clinical: Return documents and prior extractions

    Clinical->>AIMod: Request aggregation and conflict detection
    AIMod->>AIMod: Apply ACL filter — restrict to authorized patient docs (AIR-S02)
    AIMod->>VectorDB: Cosine similarity search (threshold ≥ 0.7, top-5 chunks, AIR-R02)
    VectorDB-->>AIMod: Return relevant document chunks

    AIMod->>AIMod: Re-rank chunks by semantic relevance (AIR-R03)
    AIMod->>AIMod: Construct prompt (token budget ≤ 8000, AIR-O01)

    alt Circuit Breaker OPEN (≥3 failures in 5 min — AIR-O02)
        AIMod-->>Clinical: Fallback: manual review required
        Clinical-->>Frontend: Display manual review prompt
    else Circuit Breaker CLOSED
        AIMod->>OpenAI: Send RAG prompt with grounded context
        OpenAI-->>AIMod: Structured JSON output + confidence scores
        AIMod->>AIMod: Validate output schema (≥99% validity, AIR-Q03)
        AIMod->>AIMod: Apply content filter (AIR-S04)

        loop Each extracted field
            AIMod->>AIMod: Check field confidence ≥ 80% (AIR-003)
        end

        AIMod->>Postgres: Store DataConflict records (conflicts flagged)
        AIMod->>Postgres: AuditLog — AI prompt + response (AIR-S03)
        AIMod-->>Clinical: Return aggregated view + conflicts + confidence scores
        Clinical-->>Frontend: Display 360° view with source citations and conflict flags
    end

    Staff->>Frontend: Review conflicts, select authoritative values
    Frontend->>Gateway: POST /api/conflicts/{id}/resolve
    Gateway->>Clinical: Save conflict resolution (RBAC: Staff only)
    Clinical->>Postgres: Update DataConflict.resolutionStatus

    Staff->>Frontend: Click "Verify Profile"
    Frontend->>Gateway: POST /api/patients/{id}/360view/verify
    Gateway->>Clinical: Mark profile Verified
    Clinical->>Postgres: Update 360° status = Verified
    Clinical->>Postgres: AuditLog — verification event with staff ID (FR-057, NFR-009)
    Clinical-->>Frontend: Verification confirmed
```
