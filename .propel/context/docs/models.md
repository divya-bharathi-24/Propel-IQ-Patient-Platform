## Use Case Sequence Diagrams

### UC-001: Patient Registration

**Source**: [spec.md#UC-001](.propel/context/docs/spec.md#UC-001)

```mermaid
sequenceDiagram
    participant P as Patient
    participant FE as Angular SPA
    participant GW as API Gateway
    participant AUTH as Auth Service
    participant DB as PostgreSQL
    participant EMAIL as SendGrid

    Note over P,EMAIL: UC-001 - Patient Registration

    P->>FE: Enter name, email, phone, DOB, password
    FE->>GW: POST /api/auth/register
    GW->>AUTH: Route registration request
    AUTH->>DB: Check email uniqueness
    DB-->>AUTH: Email available

    AUTH->>AUTH: Validate password complexity (8+ chars, mixed case, number, special)
    AUTH->>AUTH: Hash password (bcrypt/Argon2)
    AUTH->>DB: INSERT patient record (status: Pending)
    DB-->>AUTH: Record created

    AUTH->>EMAIL: Send verification email with token
    EMAIL-->>AUTH: Delivery accepted
    AUTH-->>GW: 201 Created - Verification email sent
    GW-->>FE: Registration success
    FE-->>P: Display "Check email for verification"

    P->>FE: Click verification link
    FE->>GW: GET /api/auth/verify?token=xxx
    GW->>AUTH: Route verification
    AUTH->>DB: UPDATE patient status to Active
    DB-->>AUTH: Status updated
    AUTH-->>GW: 200 OK - Account activated
    GW-->>FE: Redirect to login
    FE-->>P: Display login page with success message

    alt Email already registered
        AUTH->>DB: Check email uniqueness
        DB-->>AUTH: Email exists
        AUTH-->>GW: 409 Conflict
        GW-->>FE: Error response
        FE-->>P: Display "Email already registered"
    end

    alt Verification link expired (24h)
        AUTH-->>GW: 410 Gone
        GW-->>FE: Link expired
        FE-->>P: Prompt to request new verification email
    end
```

### UC-002: User Login

**Source**: [spec.md#UC-002](.propel/context/docs/spec.md#UC-002)

```mermaid
sequenceDiagram
    participant U as User (Patient/Staff/Admin)
    participant FE as Angular SPA

    participant AUTH as Auth Service
    participant DB as PostgreSQL
    participant REDIS as Upstash Redis

    Note over U,REDIS: UC-002 - User Login

    U->>FE: Enter email and password
    FE->>GW: POST /api/auth/login
    GW->>AUTH: Route login request
    AUTH->>DB: Query user by email
    DB-->>AUTH: User record (hash, role, status)

    AUTH->>AUTH: Verify password against stored hash
    AUTH->>AUTH: Generate JWT access token (15-min expiry)

    REDIS-->>AUTH: Session stored
    AUTH->>DB: Log authentication event (userId, timestamp, IP)
    AUTH-->>GW: 200 OK + JWT + refresh token
    GW-->>FE: Auth tokens
    FE->>FE: Store tokens, identify role
    FE-->>U: Redirect to role-specific dashboard

    alt Invalid credentials
        AUTH->>AUTH: Increment failed login counter
        AUTH-->>GW: 401 Unauthorized
        GW-->>FE: Generic error
        FE-->>U: Display "Invalid email or password"
    end

    alt Account deactivated
        AUTH-->>GW: 403 Forbidden
        GW-->>FE: Account inactive
        FE-->>U: Display "Contact administrator"
    end

    alt Session timeout (15-min inactivity)
        REDIS-->>AUTH: Session expired (TTL)
        FE-->>U: Redirect to login with expiration message
    end
```

```mermaid
        APT-->>GW: 409 Conflict
        GW-->>FE: Slot unavailable

        FE-->>P: Display refreshed slots
    end

        GW-->>FE: No slots
        FE-->>P: Suggest alternative dates or waitlist (UC-004)
```

**Source**: [spec.md#UC-004](.propel/context/docs/spec.md#UC-004)

```mermaid
sequenceDiagram
    participant P as Patient
    participant FE as Angular SPA
    participant GW as API Gateway
    participant APT as Appointment Service
    participant DB as PostgreSQL
    participant REDIS as Upstash Redis
    participant NOTIF as Notification Service
    participant EMAIL as SendGrid
    participant SMS as Twilio

    Note over P,SMS: UC-004 - Preferred Slot Swap

    P->>FE: Select preferred unavailable slot during booking
    FE->>GW: POST /api/waitlist
    GW->>APT: Enroll in waitlist
    APT->>DB: Create WaitlistEntry (FIFO via enrolledAt)
    DB-->>APT: Waitlist entry created
    APT-->>GW: 201 Created - Enrolled in waitlist
    GW-->>FE: Waitlist confirmation
    FE-->>P: Display "Booked alternative slot, waitlisted for preferred"

    Note over APT,DB: Triggered when preferred slot becomes available

    APT->>DB: Query waitlist for released slot (FIFO order)
    DB-->>APT: First matching WaitlistEntry
    APT->>DB: Cancel patient's original appointment
    APT->>DB: Book patient into preferred slot
    APT->>DB: Release original slot to available pool
    APT->>REDIS: Update slot cache (both slots)
    APT->>DB: Update WaitlistEntry status to Swapped

    APT->>NOTIF: Notify patient of swap
    NOTIF->>EMAIL: Send swap confirmation email
    NOTIF->>SMS: Send swap confirmation SMS
    EMAIL-->>NOTIF: Delivered
    SMS-->>NOTIF: Delivered

    alt Preferred slot claimed by direct booking first
        APT->>DB: Slot no longer available
        APT->>DB: Remove waitlist entry
        APT->>NOTIF: Notify patient waitlist expired
        NOTIF->>EMAIL: Send waitlist expiry notice
    end
```
