# User Story - US_010

## Story ID

- ID Format: US_010

## Story Title

- Patient Self-Registration with Email Verification

## Description

- As an unregistered user, I want to create a Patient account by providing my email, password, and basic demographic details — with email verification before activation — so that I can access the platform securely and my identity is confirmed.

## Acceptance Criteria

1. **Given** I am on the registration page, **When** I submit a valid email, a password meeting complexity rules (8+ chars, 1 uppercase, 1 digit, 1 special char), and demographic info, **Then** the system creates an inactive Patient account and sends a verification email within 60 seconds.

2. **Given** I receive the verification email, **When** I click the verification link, **Then** my account is activated (`emailVerified = true`), I am redirected to the booking interface, and a registration audit event is logged with UTC timestamp, user ID, and IP address.

3. **Given** I try to register with an email already in use, **When** I submit the form, **Then** the system returns an error message "Email already registered" without revealing whether the account is active or inactive.

4. **Given** I submit a password that does not meet complexity rules, **When** the form is validated, **Then** each violated rule is displayed inline (e.g., "Must include at least one special character") without exposing internal error details.

## Edge Cases

- What happens if I do not verify my email within 24 hours? (Verification token expires; I can request a new verification email from the login page)
- What happens if the verification link is clicked twice? (Second click returns "Link already used" with a prompt to log in)
