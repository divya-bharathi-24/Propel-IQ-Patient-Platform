# User Story - US_011

## Story ID

- ID Format: US_011

## Story Title

- User Login, JWT Tokens & Session Auto-Timeout

## Description

- As an authenticated user (Patient, Staff, or Admin), I want to log in with my credentials and receive a 15-minute JWT access token backed by a Redis session, so that my identity is securely verified and my session automatically expires after 15 minutes of inactivity.

## Acceptance Criteria

1. **Given** I enter valid credentials on the login page, **When** I submit, **Then** I receive a JWT access token (15-min expiry) and a rotating refresh token, and a Redis session entry is created with a 15-minute TTL.

2. **Given** my session has been inactive for exactly 15 minutes, **When** the Redis TTL expires, **Then** the next API call returns HTTP 401 and I am redirected to the login page with a session expired message.

3. **Given** I am authenticated, **When** my access token expires but my session is still active, **Then** the refresh token endpoint silently issues a new access token and rotates the refresh token without requiring re-login.

4. **Given** I click "Logout", **When** the logout action executes, **Then** my Redis session is deleted immediately, the refresh token is invalidated, and a logout audit event is logged.

## Edge Cases

- What happens if someone steals my refresh token? (Refresh token rotation: reuse of an already-rotated token invalidates the entire session and logs a security alert)
- What happens when I log in from multiple devices simultaneously? (Each device gets its own session; all sessions respect the 15-minute inactivity rule independently)
