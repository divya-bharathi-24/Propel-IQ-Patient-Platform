# User Story - US_012

## Story ID

- ID Format: US_012

## Story Title

- Admin-Managed Staff & Admin Account Creation

## Description

- As an Admin, I want to create Staff and Admin accounts so that new team members can be onboarded with correct role assignments, and as a Staff member I want to create a basic Patient account during walk-in so that walk-in visits are properly registered.

## Acceptance Criteria

1. **Given** I am an authenticated Admin, **When** I create a new Staff account with a name, email, and role, **Then** the account is created with `status = Active`, a credential setup email is sent to the new user, and the action is logged in the audit trail.

2. **Given** a new Staff/Admin account is created, **When** the new user opens the credential setup email, **Then** they are directed to a one-time password setup page where they set a compliant password (8+ chars, mixed case, digit, special char).

3. **Given** I am a Staff member in the walk-in booking flow, **When** I choose to create a Patient account, **Then** I can enter name, contact number, and email — resulting in a basic Patient record linked to the walk-in appointment.

4. **Given** I am a non-Admin user, **When** I attempt to access the account creation endpoints, **Then** I receive HTTP 403 Forbidden and no account is created.

## Edge Cases

- What happens if the credential setup email bounces? (Admin is notified of delivery failure; can resend from the user management interface)
- What happens if a Staff member creates a Patient account with a duplicate email? (System returns a validation error indicating the email is already registered and offers to link to the existing account)
