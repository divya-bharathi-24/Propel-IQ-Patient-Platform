# User Story - US_013

## Story ID

- ID: US_013

## Story Title

- Session Timeout and Auth Event Logging

## Description

- As a platform user, I want my session to automatically terminate after 15 minutes of inactivity and have all authentication events logged, so that unauthorized access is prevented through idle session expiry and all auth activity is auditable for HIPAA compliance.

## Acceptance Criteria

1. **Given** I am authenticated, **When** I am inactive for 15 minutes, **Then** the system terminates my session via Redis TTL expiry and redirects me to the login page with a "Session expired" message.

2. **Given** I am interacting with the platform, **When** I perform an action before the 15-minute timeout, **Then** the session TTL is reset to 15 minutes from the last activity.

3. **Given** any user attempts to log in, **When** the login succeeds, **Then** the system logs the event with userId, timestamp, source IP, and action type "Login".

4. **Given** any user is authenticated, **When** they log out, **Then** the system logs the event with userId, timestamp, and action type "Logout" and invalidates the session in Redis.

5. **Given** any user attempts to log in, **When** the login fails, **Then** the system logs the event with timestamp, source IP, attempted email, and action type "FailedLogin".

6. **Given** a session expires, **When** the timeout occurs, **Then** the system logs the event with userId, timestamp, and action type "SessionTimeout".

## Edge Cases

- What happens when multiple browser tabs are open? (Session is shared via token; activity in any tab resets the timeout; expiry in Redis affects all tabs simultaneously)
- How does the system handle clock skew between client and server? (Server-side Redis TTL is authoritative; client-side countdown is cosmetic guidance only)
