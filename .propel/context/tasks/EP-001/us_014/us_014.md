# User Story - US_014

## Story ID

- ID: US_014

## Story Title

- Admin User Management Panel

## Description

- As an Admin, I want to create, update, and deactivate user accounts for all roles (Patient, Staff, Admin) through a management panel, so that I can control platform access and manage the user lifecycle centrally.

## Acceptance Criteria

1. **Given** I am authenticated as Admin, **When** I navigate to the user management panel, **Then** I see a list of all users with name, email, role, and status columns with search and filter capabilities.

2. **Given** I am on the user management panel, **When** I create a new user by entering name, email, and role, **Then** the system creates the account, sends an invitation email, and logs the action in the audit trail.

3. **Given** I am creating a user, **When** I enter an email that already exists in the system, **Then** the system displays an error "Email already registered" and prevents duplicate creation.

4. **Given** I am on the user management panel, **When** I update a user's details (name, role, contact info), **Then** the system saves the changes and logs the update action in the audit trail.

5. **Given** I am on the user management panel, **When** I deactivate a user account, **Then** the system sets the account status to Deactivated, terminates any active sessions in Redis, and logs the deactivation in the audit trail.

6. **Given** I am attempting to deactivate an account, **When** I select my own account, **Then** the system prevents the action and displays "Cannot deactivate your own account".

## Edge Cases

- What happens when an Admin changes a user's role from Staff to Patient? (Role change takes effect on next token refresh; existing session permissions are revoked)
- How does the system handle deactivation of a user with upcoming appointments? (Appointments remain active; staff is notified to handle affected appointments)
