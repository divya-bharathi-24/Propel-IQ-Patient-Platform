# User Story - US_016

## Story ID

- ID: US_016

## Story Title

- Appointment PDF Generation and Email

## Description

- As a patient who has booked an appointment, I want to receive a PDF with appointment details (date, time, provider, location, instructions) sent to my registered email, so that I have a portable reference document for my upcoming visit.

## Acceptance Criteria

1. **Given** I have successfully booked an appointment, **When** the booking is confirmed, **Then** the system generates a PDF containing appointment date, time, provider specialty, location, and patient instructions using QuestPDF.

2. **Given** the PDF is generated, **When** the system sends the email, **Then** the PDF is delivered to my registered email address as an attachment within 2 minutes of booking.

3. **Given** the email is sent, **When** I open the PDF, **Then** the document is well-formatted, readable, and contains accurate appointment details matching my booking.

4. **Given** the email delivery fails, **When** the system detects the failure, **Then** it logs the failure and displays a download link for the PDF on the booking confirmation page as a fallback.

5. **Given** I reschedule an appointment, **When** the reschedule is confirmed, **Then** the system generates an updated PDF with new appointment details and sends it to my email.

## Edge Cases

- What happens when the PDF generation fails due to invalid data? (System logs error; booking confirmation page provides appointment details in text format as fallback)
- How does the system handle email bounces for invalid email addresses? (Delivery failure is logged; notification record tracks the failure; PDF remains downloadable from profile)
