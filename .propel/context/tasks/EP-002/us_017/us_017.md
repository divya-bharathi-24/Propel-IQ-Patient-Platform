# User Story - US_017

## Story ID

- ID Format: US_017

## Story Title

- Patient Self-Service Intake Edit Without Duplicate Records

## Description

- As an authenticated Patient, I want to review and update my previously submitted intake information at any time without staff assistance, so that my health history stays accurate and no duplicate intake records are created.

## Acceptance Criteria

1. **Given** I have a previously submitted intake record, **When** I navigate to "Edit Intake" from the dashboard, **Then** all previously saved fields are pre-populated with my last submitted values.

2. **Given** I modify an intake field and save, **When** the save operation completes, **Then** the existing IntakeRecord is updated (UPSERT) — no new IntakeRecord row is created — and the `completedAt` timestamp is updated.

3. **Given** I edit intake without completing all required fields, **When** I attempt to save, **Then** the system saves the partial update as a draft and displays which fields remain incomplete.

4. **Given** I resume editing an intake after a session timeout, **When** I return to the edit form, **Then** my draft values from before the timeout are restored from the saved draft state.

## Edge Cases

- What happens if staff submits an intake on my behalf at the same time I am editing? (Optimistic concurrency; the last submitter receives a conflict warning and is shown the other version for reconciliation)
- What happens when I switch from AI-assisted intake mode to manual mode after editing? (All data entered in AI mode is transferred to the manual form fields; no data is lost)
