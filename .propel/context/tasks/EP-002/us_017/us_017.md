# User Story - US_017

## Story ID

- ID: US_017

## Story Title

- Appointment Cancel and Reschedule

## Description

- As a patient or staff member, I want to cancel an existing appointment with a reason or reschedule it to a new available slot, so that appointment changes are managed efficiently with proper slot release and audit tracking.

## Acceptance Criteria

1. **Given** I have an active appointment, **When** I cancel it and provide a cancellation reason, **Then** the system updates the appointment status to "Cancelled", stores the reason, and releases the time slot back to the available pool.

2. **Given** an appointment is cancelled, **When** the slot is released, **Then** the system checks the waitlist for the released slot and triggers a preferred slot swap if applicable.

3. **Given** I have an active appointment, **When** I reschedule by selecting a new available time slot, **Then** the system moves the appointment to the new slot, releases the original slot, and sends an updated appointment PDF to my email.

4. **Given** I am rescheduling, **When** no alternative slots are available for my selected criteria, **Then** the system suggests waitlist enrollment for a preferred slot.

5. **Given** a Staff member cancels on behalf of a patient, **When** the cancellation is processed, **Then** the system logs the staff user who performed the action in the audit trail.

6. **Given** an appointment is cancelled or rescheduled, **When** the operation completes, **Then** the patient receives an email notification with the updated appointment status.

## Edge Cases

- What happens when a patient tries to cancel an appointment already marked as "Arrived"? (System prevents cancellation; displays "Cannot cancel an appointment after check-in")
- How does the system handle rescheduling to a slot that becomes unavailable during selection? (Optimistic locking detects conflict; system refreshes available slots)
