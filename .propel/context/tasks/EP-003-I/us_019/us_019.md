# User Story - US_019

## Story ID

- ID Format: US_019

## Story Title

- End-to-End Single-Session Appointment Booking Workflow

## Description

- As an authenticated Patient, I want to complete an appointment booking in a single end-to-end session (slot selection, intake mode selection, insurance pre-check, confirmation) without needing to return later, so that I can confirm my appointment quickly and confidently.

## Acceptance Criteria

1. **Given** I have selected an available slot, **When** I proceed through the booking wizard (slot → intake mode → insurance → confirm), **Then** all four steps are completable in a single browser session without being redirected away.

2. **Given** I submit the booking, **When** the system commits the appointment, **Then** an Appointment record is created with `status = Booked`, an InsuranceValidation record is stored, and a WaitlistEntry is created if I designated a preferred slot.

3. **Given** two patients submit bookings for the same slot at the exact same moment, **When** the system processes both requests, **Then** exactly one succeeds and the other receives an HTTP 409 Conflict with a "Slot no longer available" message.

4. **Given** my booking is confirmed, **When** I view the confirmation screen, **Then** I see the appointment date, time, provider specialty, reference number, and a "Add to Calendar" option.

## Edge Cases

- What happens if I close the browser at the intake mode selection step before confirming? (The slot reservation is released after a 5-minute hold period; no orphaned booking is left)
- What happens if the insurance pre-check service is temporarily unavailable? (Booking proceeds; insurance status is displayed as "Check Pending" and retried asynchronously)
