# User Story - US_021

## Story ID

- ID Format: US_021

## Story Title

- Branded PDF Confirmation Email Within 60 Seconds

## Description

- As a Patient, I want to receive a branded PDF confirmation email within 60 seconds of booking confirmation, so that I have a physical record of my appointment with all the details I need.

## Acceptance Criteria

1. **Given** my booking is confirmed, **When** the booking confirmation event fires, **Then** a PDF is generated via QuestPDF containing appointment date, start time, end time, provider specialty, clinic/location name, and booking reference number.

2. **Given** the PDF is generated, **When** it is emailed via SendGrid, **Then** the email is delivered to my registered email address within 60 seconds of booking confirmation.

3. **Given** the email delivery attempt is made, **When** SendGrid returns a delivery status, **Then** the Notification record is updated with `status = Sent` (or `Failed`) and the delivery event is logged in the audit trail.

4. **Given** the initial PDF generation fails, **When** the retry job runs, **Then** the system retries generation once within 2 minutes and delivers the PDF; if the retry also fails, a failure notification is logged and the patient is notified via a dashboard alert.

## Edge Cases

- What happens if the patient's email address is invalid? (SendGrid returns a bounce; the notification status is set to `Failed`; a dashboard alert prompts the patient to update their email)
- What happens if QuestPDF throws an exception on generation? (Exception is caught, logged with the appointment ID, and the job is queued for immediate retry)
