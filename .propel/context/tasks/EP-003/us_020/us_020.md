# User Story - US_020

## Story ID

- ID: US_020

## Story Title

- Manual Form Patient Intake

## Description

- As a patient, I want to complete my intake via a traditional manual form with fields for demographics, medical history, symptoms, and medications, so that I have a structured alternative to AI conversational intake.

## Acceptance Criteria

1. **Given** I have a booked appointment, **When** I initiate the intake process and select manual form mode, **Then** the system displays a structured intake form with all required fields (demographics, medical history, symptoms, medications).

2. **Given** I am filling the form, **When** I submit with all required fields completed and valid, **Then** the system saves the IntakeRecord with source indicator "Manual" linked to my patient profile and appointment.

3. **Given** I am filling the form, **When** I submit with missing required fields or invalid data formats, **Then** the system highlights the invalid fields with specific error messages and prevents submission.

4. **Given** I have submitted the form, **When** the intake is saved, **Then** I see a confirmation message and the data is available for clinical processing.

5. **Given** I am filling the form, **When** I partially complete fields and navigate away, **Then** the system warns me about unsaved data before leaving the page.

## Edge Cases

- What happens when the form submission fails due to a server error? (System displays retry option; form data is preserved in the browser to prevent data loss)
- How does the system handle extremely long text entries in medical history? (Input fields have configurable character limits with clear indicators; overflow is truncated with warning)
