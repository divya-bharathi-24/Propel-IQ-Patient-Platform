# User Story - US_021

## Story ID

- ID: US_021

## Story Title

- Intake Mode Switching and Self-Review

## Description

- As a patient, I want to switch between AI conversational intake and manual form intake at any point without losing data, and review and edit all my intake responses, so that I have full flexibility and control over how I provide my health information.

## Acceptance Criteria

1. **Given** I am in AI conversational intake, **When** I request to switch to manual form, **Then** the system transfers all collected AI data into the corresponding manual form fields and I can continue from where I left off.

2. **Given** I am in manual form intake, **When** I request to switch to AI conversational mode, **Then** the system transfers all entered form data to the AI session context and the AI continues the conversation from the remaining uncollected fields.

3. **Given** I have completed intake via either mode, **When** I access the review screen, **Then** the system displays all my responses organized by category with edit capability for each field.

4. **Given** I am reviewing my intake responses, **When** I edit a specific field, **Then** the system updates only that field without requiring re-submission of the entire intake form.

5. **Given** data has been partially collected in one mode, **When** I switch modes, **Then** zero data loss occurs and all previously provided information is accurately preserved.

## Edge Cases

- What happens when switching modes during an AI multi-turn question sequence? (Partial response for the current question is discarded; all previously confirmed answers are transferred)
- How does the system handle data format differences between AI-extracted and manual form entries? (Normalization layer standardizes data formats before transfer between modes)
