# User Story - US_024

## Story ID

- ID: US_024

## Story Title

- Same-Day Queue Management

## Description

- As a Staff member, I want to manage a same-day appointment queue including adding, reordering, and removing patients, so that I can efficiently control patient flow and prioritize cases throughout the day.

## Acceptance Criteria

1. **Given** I am authenticated as Staff, **When** I view the same-day queue dashboard, **Then** the system displays the current queue with patient names, queue positions, arrival times, and statuses in real-time.

2. **Given** I add a walk-in patient to the queue, **When** the patient is added, **Then** the system assigns a queue position based on arrival order and creates a QueueEntry record with status "Waiting".

3. **Given** the queue has multiple patients, **When** I reorder a patient to a higher priority position, **Then** the system adjusts all subsequent queue positions accordingly and updates the display in real-time.

4. **Given** a patient is in the queue, **When** I remove them (called or left), **Then** the system updates the QueueEntry status, adjusts remaining positions, and logs the event in the audit trail.

5. **Given** the queue state changes, **When** add, reorder, or remove operations occur, **Then** the Redis-cached queue state is updated within 5 seconds and the dashboard reflects the change.

## Edge Cases

- What happens when two staff members simultaneously reorder the same queue? (Optimistic locking prevents conflicting updates; second operation receives conflict notification and refreshes)
- How does the system handle a queue with no patients? (Dashboard displays empty state message "No patients in queue")
