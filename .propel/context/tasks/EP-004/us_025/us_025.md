# User Story - US_025

## Story ID

- ID: US_025

## Story Title

- Staff Appointment Arrival Marking

## Description

- As a Staff member, I want to mark a patient's appointment as "Arrived" from the daily schedule dashboard, so that the clinic can track patient arrivals and manage the appointment flow accurately.

## Acceptance Criteria

1. **Given** I am authenticated as Staff, **When** I view today's appointment list, **Then** the system displays all appointments for the current date with patient names, times, specialties, and current statuses.

2. **Given** I locate a patient's appointment, **When** I mark it as "Arrived", **Then** the system updates the appointment status to "Arrived" and records the arrival timestamp.

3. **Given** I mark an appointment as arrived, **When** the action completes, **Then** the system creates an audit log entry with my staff ID, action type "MarkArrived", and timestamp.

4. **Given** a patient is not found in today's appointment list, **When** I search for them, **Then** the system prompts me to verify appointment details or create a walk-in booking.

5. **Given** I am a Patient user, **When** I attempt to mark an appointment as arrived, **Then** the system returns a 403 Forbidden response (Staff-only action).

## Edge Cases

- What happens when a staff member tries to mark an already-arrived appointment? (System displays "Already marked as Arrived" and prevents duplicate marking)
- How does the system handle arrival marking after the appointment time has passed? (Allow marking with a "Late Arrival" indicator recorded in the appointment details)
