# User Story - US_027

## Story ID

- ID Format: US_027

## Story Title

- Same-Day Queue View & Arrived Status Marking

## Description

- As an authenticated Staff member, I want to view all appointments for today in a real-time queue with patient names, times, booking types, and arrival statuses, and mark patients as arrived with a timestamp, so that I can efficiently manage the daily patient flow.

## Acceptance Criteria

1. **Given** I navigate to the same-day queue view, **When** the page loads, **Then** all appointments for the current calendar day are displayed with patient name, appointment time, booking type (Self-Booked / Walk-In), and current arrival status (Waiting / Arrived / Cancelled).

2. **Given** I click "Mark as Arrived" for a patient, **When** the action is confirmed, **Then** the Appointment status is updated to `Arrived`, the arrival timestamp is recorded (UTC), the QueueEntry is updated to `Called`, and the audit log records the action with my staff ID.

3. **Given** a new walk-in is added to today's queue after the page has loaded, **When** the queue refreshes (polling or real-time), **Then** the new entry appears in the queue within 10 seconds without requiring a full page reload.

4. **Given** a Patient-role user attempts to mark themselves as arrived via any endpoint, **When** the request is evaluated, **Then** HTTP 403 Forbidden is returned and no status change occurs.

## Edge Cases

- What happens if I accidentally mark the wrong patient as arrived? (Arrived status can be reversed to "Waiting" by Staff within the same session day only)
- What happens when the queue has 0 appointments for today? (Empty state is shown with a message "No appointments scheduled for today")
