# User Story - US_026

## Story ID

- ID: US_026

## Story Title

- Automated SMS Appointment Reminders

## Description

- As a patient with an upcoming appointment, I want to receive automated SMS reminders at configurable intervals before my appointment via Twilio, so that I am reminded of my visit and less likely to miss it.

## Acceptance Criteria

1. **Given** I have an active booked appointment, **When** the configured reminder interval is reached (e.g., 24 hours before), **Then** the system sends an SMS reminder to my registered phone number containing appointment date, time, and provider specialty.

2. **Given** the SMS is sent, **When** delivery status is returned by Twilio, **Then** the system creates a Notification record with channel "SMS", delivery status, timestamp, and retry count.

3. **Given** SMS delivery fails, **When** the failure is detected, **Then** the system retries once and logs both attempts; if retry fails, the system relies on the email reminder channel.

4. **Given** my appointment is cancelled before the reminder trigger time, **When** the scheduler checks for reminders, **Then** the system skips reminder generation for the cancelled appointment.

5. **Given** the Twilio service is unavailable, **When** the system attempts to send SMS, **Then** it gracefully degrades by logging the failure and relying on email reminders without blocking the workflow.

## Edge Cases

- What happens when a patient has no phone number on file? (System skips SMS channel; relies solely on email reminders; notification record notes "No phone number")
- How does the system handle timezone differences for reminder scheduling? (All times stored in UTC; reminders calculated based on appointment timezone)
