# User Story - US_027

## Story ID

- ID: US_027

## Story Title

- Automated Email Appointment Reminders

## Description

- As a patient with an upcoming appointment, I want to receive automated email reminders at configurable intervals before my appointment via SendGrid, so that I have email confirmation of my upcoming visit with all relevant details.

## Acceptance Criteria

1. **Given** I have an active booked appointment, **When** the configured reminder interval is reached (e.g., 24 hours, 2 hours before), **Then** the system sends an email reminder to my registered email address with appointment date, time, provider specialty, and location.

2. **Given** the email is sent, **When** delivery status is returned by SendGrid, **Then** the system creates a Notification record with channel "Email", delivery status, timestamp, and retry count.

3. **Given** email delivery fails, **When** the failure is detected, **Then** the system retries once and logs both attempts; if retry fails, the system relies on the SMS reminder channel.

4. **Given** my appointment is cancelled before the reminder trigger time, **When** the scheduler checks for reminders, **Then** the system skips reminder generation for the cancelled appointment.

5. **Given** the SendGrid service is unavailable, **When** the system attempts to send email, **Then** it gracefully degrades by logging the failure and relying on SMS reminders without blocking the workflow.

## Edge Cases

- What happens when the patient's email address bounces? (Delivery failure is logged; notification record tracks bounce; SMS channel is the fallback)
- How does the system handle reminder deduplication? (Unique constraint on appointmentId + channel + interval prevents duplicate reminders for the same trigger)
