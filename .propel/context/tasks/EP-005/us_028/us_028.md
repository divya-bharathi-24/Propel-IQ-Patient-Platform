# User Story - US_028

## Story ID

- ID Format: US_028

## Story Title

- AI Conversational Intake Chat Interface

## Description

- As a Patient completing intake, I want an AI-assisted chat interface that dynamically guides me through health history questions and auto-populates structured intake fields from my free-text responses, so that I can provide my health information conversationally without filling out a long form.

## Acceptance Criteria

1. **Given** I select "AI-Assisted" intake mode, **When** the chat session starts, **Then** the AI sends an opening contextual question and a live form preview panel shows the fields that will be auto-populated.

2. **Given** I type a free-text response, **When** the AI processes it using NLU, **Then** the structured intake fields (demographics, medical history, symptoms, medications) are auto-populated in the live preview, and the next question is generated within 5 seconds.

3. **Given** a field is extracted with confidence below 80%, **When** the field is populated in the preview, **Then** it is flagged with a visual indicator and I am asked a clarifying follow-up question for that specific field.

4. **Given** I complete the multi-turn conversation, **When** I review and confirm the auto-populated fields, **Then** submitting the intake saves an IntakeRecord with `source = AI` and all confirmed field values stored in the JSONB columns.

## Edge Cases

- What happens if the OpenAI API is unavailable during intake? (Circuit breaker falls back to manual form mode; data entered in the AI chat is preserved and pre-populated in the manual form)
- What happens if I provide a response that is too short for NLU extraction? (AI asks a more specific follow-up question; the field is not populated until a sufficient response is given)
