# User Story - US_028

## Story ID

- ID: US_028

## Story Title

- Staff Reminder Configuration

## Description

- As a Staff member, I want to configure reminder timing rules including intervals and channels at the system level, so that appointment reminders are sent at appropriate times through the right channels for our clinic's workflow.

## Acceptance Criteria

1. **Given** I am authenticated as Staff, **When** I navigate to the reminder configuration settings, **Then** I see the current reminder rules including intervals (e.g., 24h, 2h) and channels (SMS, Email) with ability to modify.

2. **Given** I am on the configuration screen, **When** I add a new reminder interval (e.g., 48 hours before), **Then** the system saves the new rule and future appointments use the updated intervals.

3. **Given** I am configuring reminders, **When** I enable or disable a specific channel (SMS or Email) for a specific interval, **Then** the system updates the configuration and applies it to all future reminder triggers.

4. **Given** the configuration is updated, **When** the external notification services (Twilio, SendGrid) are unavailable, **Then** the system falls back to the available channel and provides an ICS file download option for patients.

5. **Given** reminder configuration exists, **When** an appointment reminder is triggered, **Then** the system uses the current system-level configuration to determine which channels and intervals apply.

## Edge Cases

- What happens when all notification channels are disabled? (System warns staff that no reminders will be sent; configuration saved with warning flag)
- How does the system handle configuration changes for already-scheduled reminders? (Changes apply to future triggers only; already-queued reminders execute with previous configuration)
