# User Story - US_029

## Story ID

- ID: US_029

## Story Title

- Google Calendar Sync

## Description

- As a patient, I want to sync my booked appointments to my Google Calendar via OAuth 2.0 authorization, so that my healthcare appointments appear alongside my other events and I receive native calendar reminders.

## Acceptance Criteria

1. **Given** I have a booked appointment, **When** I select "Sync to Google Calendar", **Then** the system initiates the Google OAuth 2.0 authorization flow if I have not previously authorized.

2. **Given** I authorize the application, **When** the OAuth flow completes, **Then** the system securely stores the access/refresh tokens and creates a calendar event with appointment date, time, provider specialty, and location.

3. **Given** I have previously authorized Google Calendar access, **When** I sync a new appointment, **Then** the system skips the authorization step and directly creates the calendar event.

4. **Given** the calendar event is created, **When** the sync completes, **Then** the system creates a CalendarSync record with provider "Google", external event ID, syncStatus "Synced", and displays confirmation to me.

5. **Given** the Google Calendar API rate limit is exceeded, **When** the sync fails, **Then** the system queues the request and retries with exponential backoff, logging the delay.

6. **Given** my authorization is revoked or expired, **When** I attempt to sync, **Then** the system prompts me to re-authorize calendar access.

## Edge Cases

- What happens when the patient revokes Google access after syncing? (CalendarSync record updated to "Revoked"; next sync attempt prompts re-authorization)
- How does the system handle appointment updates after calendar sync? (Phase 1: new sync creates/updates event; cancelled appointments trigger event deletion if authorized)
