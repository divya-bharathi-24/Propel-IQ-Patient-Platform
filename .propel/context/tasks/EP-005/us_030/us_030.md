# User Story - US_030

## Story ID

- ID Format: US_030

## Story Title

- Intake Mode Switch, Autosave & Resume

## Description

- As a Patient, I want to switch between AI-assisted and manual intake modes at any point, with all my progress autosaved, so that I never lose entered data and can return to finish my intake from where I left off.

## Acceptance Criteria

1. **Given** I am in AI-assisted intake mode and click "Switch to Manual Form", **When** the mode switch executes, **Then** all AI-extracted field values are transferred to the manual form fields instantly with no data loss.

2. **Given** I am in manual form mode and click "Switch to AI Mode", **When** the mode switch executes, **Then** the AI chat is initialized with a context summary of fields already filled in the manual form, and I continue from the next unfilled question.

3. **Given** I navigate away from the intake page without submitting, **When** I return to the intake page on the same or a later session, **Then** my last autosaved draft is restored automatically with a "Resuming your saved intake" notification.

4. **Given** autosave is active, **When** I modify any field, **Then** an autosave is triggered within 30 seconds and the page shows a "Saved" indicator confirming the draft was stored.

## Edge Cases

- What happens if autosave fails due to a network interruption? (A local browser storage backup is maintained; on reconnect, the local draft is synced to the server with a conflict check)
- What happens when switching modes resets a field that was manually corrected in one mode? (The most recently edited value for each field is preserved regardless of which mode last modified it)
