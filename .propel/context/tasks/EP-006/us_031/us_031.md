# User Story - US_031

## Story ID

- ID: US_031

## Story Title

- Clinical Document Upload and Validation

## Description

- As a patient, I want to upload historical clinical documents in PDF format with file validation, so that my clinical records are securely stored and queued for AI-powered data extraction.

## Acceptance Criteria

1. **Given** I am authenticated as a Patient, **When** I navigate to the document upload section, **Then** the system displays an upload interface accepting PDF files with clear format and size guidance.

2. **Given** I select PDF files for upload, **When** the files pass format (PDF only) and size validation (up to 50 MB), **Then** the system uploads and stores the documents securely with AES-256 encryption at rest.

3. **Given** the document is uploaded, **When** storage completes, **Then** the system creates a ClinicalDocument record with status "Pending" and queues the document for AI extraction processing.

4. **Given** the document is queued, **When** I view the upload confirmation, **Then** the system displays a processing status indicator showing the current state (Pending, Processing, Completed, Failed).

5. **Given** I upload a non-PDF file, **When** validation runs, **Then** the system rejects the upload and displays "Only PDF files are supported" with no file stored.

6. **Given** I upload a file exceeding 50 MB, **When** validation runs, **Then** the system rejects the upload and displays the maximum file size limit.

## Edge Cases

- What happens when a PDF is corrupt or password-protected? (System accepts upload but marks extraction status as "Failed" with error details during processing)
- How does the system handle concurrent uploads from the same patient? (Each upload is processed independently; multiple ClinicalDocument records created)
