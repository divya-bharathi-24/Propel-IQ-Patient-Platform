# User Story - US_032

## Story ID

- ID: US_032

## Story Title

- AI-Powered Clinical Data Extraction

## Description

- As a clinical staff member, I want the system to automatically extract structured patient data (vitals, medical history, medications, allergies, diagnoses) from uploaded PDF clinical documents with source citations and confidence scoring, so that I can verify extracted data in minutes instead of manually reading 20-minute documents.

## Acceptance Criteria

1. **Given** a PDF document is queued for processing, **When** the AI extraction pipeline runs, **Then** the system extracts structured data (vitals, medical history, medications, allergies, diagnoses) and stores each data point as an ExtractedData record with per-field confidence scores.

2. **Given** data is extracted, **When** an extraction result is displayed, **Then** each data point includes a source citation with document name, page number, and extracted text snippet for Trust-First verification.

3. **Given** the AI extraction produces a confidence score below 80% for any field, **When** the result is processed, **Then** the system automatically flags that field for manual staff review and sets the document to require staff verification.

4. **Given** the document is processed, **When** extraction completes, **Then** the system updates the ClinicalDocument processing status to "Completed" and the patient's 360-degree view is queued for update.

5. **Given** the AI extraction runs, **When** it completes, **Then** the total processing time is within 30 seconds (p95 latency) with a progress indicator displayed for operations exceeding 5 seconds.

## Edge Cases

- What happens when the PDF contains scanned images rather than text? (OCR is attempted on best-effort basis; lower confidence scores are expected; document flagged for manual review)
- How does the system handle a document with conflicting data within itself? (Each data point is extracted individually with source citation; conflicts are identified during 360-view aggregation)
