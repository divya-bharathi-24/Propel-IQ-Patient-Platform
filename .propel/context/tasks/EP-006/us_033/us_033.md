# User Story - US_033

## Story ID

- ID: US_033

## Story Title

- Post-Visit Clinical Notes Processing

## Description

- As a clinical staff member, I want the system to ingest and process post-visit clinical notes to extract relevant patient data and update the consolidated patient record, so that post-visit documentation is automatically incorporated into the patient's 360-degree view.

## Acceptance Criteria

1. **Given** a post-visit clinical note is uploaded or ingested, **When** the system processes it, **Then** relevant patient data (vitals, diagnoses, medications, treatment notes) is extracted using the AI extraction pipeline with per-field confidence scores.

2. **Given** the clinical note is processed, **When** extraction completes, **Then** the extracted data is linked to the source document with source citations (document name, page number, text snippet).

3. **Given** extracted data from the clinical note, **When** the patient's consolidated record is updated, **Then** new data points are merged into the 360-degree patient view with source-linked traceability.

4. **Given** the clinical note extraction produces data that conflicts with existing patient data, **When** the conflict is detected, **Then** a DataConflict record is created for staff resolution.

5. **Given** the post-visit note extraction runs, **When** confidence is below 80% for any field, **Then** the field is flagged for manual staff review before being committed to the patient record.

## Edge Cases

- What happens when a post-visit note contains data for multiple patients? (System processes data for the linked patient only; unrecognized patient references are logged and flagged)
- How does the system handle duplicate post-visit notes for the same visit? (Deduplication check by document hash; duplicate uploads are flagged with reference to existing document)
