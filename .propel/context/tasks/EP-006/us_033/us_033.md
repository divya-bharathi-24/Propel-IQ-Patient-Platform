# User Story - US_033

## Story ID

- ID Format: US_033

## Story Title

- Automated Multi-Channel Reminders with Configurable Intervals

## Description

- As a Patient with a confirmed appointment, I want to receive automated email and SMS reminders at 48 hours, 24 hours, and 2 hours before my appointment — with intervals configurable by Staff — so that I am well-reminded and less likely to forget my appointment.

## Acceptance Criteria

1. **Given** an appointment is confirmed with status `Booked`, **When** the reminder scheduler evaluates upcoming appointments, **Then** reminder jobs are queued for the 48h, 24h, and 2h windows before the appointment time.

2. **Given** a reminder job fires at the scheduled time, **When** the notification is dispatched, **Then** both an email (via SendGrid) and an SMS (via Twilio) are sent with the patient's name, appointment date, time, provider, and reference number.

3. **Given** a Staff or Admin user changes the default reminder intervals in system settings, **When** the new intervals are saved, **Then** future reminder schedules are recalculated using the new intervals; already-queued reminders are updated if the appointment time has not passed.

4. **Given** an appointment is cancelled before a scheduled reminder fires, **When** the scheduler evaluates the appointment, **Then** the reminder is suppressed and a suppression event is logged in the Notification record.

## Edge Cases

- What happens if the patient's phone number is invalid for SMS delivery? (SMS send attempt is made; SendGrid/Twilio returns an error; Notification is logged as `Failed`; email delivery is the fallback)
- What happens if the reminder job server crashes mid-execution? (Jobs are persisted to the database queue; on restart, incomplete jobs are retried once with at-least-once delivery semantics)
