# User Story - US_034

## Story ID

- ID Format: US_034

## Story Title

- Manual Ad-Hoc Reminder Trigger & Delivery Logging

## Description

- As an authenticated Staff member, I want to manually trigger an immediate reminder for any specific patient appointment from the appointment detail view, so that I can send an urgent nudge for high-risk or unresponsive patients independent of the automated schedule.

## Acceptance Criteria

1. **Given** I am viewing a specific appointment detail, **When** I click "Send Reminder Now", **Then** an immediate email and SMS reminder is dispatched for that appointment containing all required appointment details.

2. **Given** the manual reminder is dispatched, **When** the delivery statuses are received from SendGrid and Twilio, **Then** a Notification record is created for each channel with `status = Sent/Failed`, the UTC timestamp, and `triggeredBy = staffId`.

3. **Given** I manually trigger a reminder, **When** the action completes, **Then** the appointment detail view shows a "Last reminder sent: [timestamp] by [staff name]" confirmation.

4. **Given** a reminder delivery fails, **When** I view the Notification record, **Then** the failure is logged with the error reason from the delivery provider and I can retry from the UI.

## Edge Cases

- What happens if I trigger a manual reminder for a cancelled appointment? (System rejects the action with a validation error: "Cannot send reminders for cancelled appointments")
- What happens when multiple Staff members trigger reminders for the same appointment in quick succession? (A debounce cooldown of 5 minutes prevents duplicate sends; subsequent attempts show "Reminder recently sent — retry available in X minutes")
