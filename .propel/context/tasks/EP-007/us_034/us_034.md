# User Story - US_034

## Story ID

- ID: US_034

## Story Title

- 360-Degree Patient View Aggregation

## Description

- As a Staff member, I want to view a consolidated, de-duplicated 360-degree patient profile that aggregates all uploaded documents and processed clinical notes with source-linked data points, so that I have a comprehensive view of the patient's clinical history in one place.

## Acceptance Criteria

1. **Given** I am authenticated as Staff, **When** I select a patient and open their 360-degree view, **Then** the system displays aggregated data sections for vitals, medical history, medications, allergies, and diagnoses from all uploaded documents and intake records.

2. **Given** the 360-degree view is displayed, **When** I view a data point, **Then** the source document name and extraction confidence are indicated alongside the value.

3. **Given** multiple documents contain the same data type, **When** the aggregation runs, **Then** the system de-duplicates identical values and presents the most recent or highest-confidence value as primary.

4. **Given** the document chunks are processed, **When** the RAG pipeline retrieves relevant data, **Then** documents are chunked into 512-token segments with 10% overlap and top-5 relevant chunks are retrieved with cosine similarity threshold of 0.7 or higher.

5. **Given** unresolved data conflicts exist, **When** the 360-degree view loads, **Then** the system displays a conflict alert banner with visual indicators highlighting conflicting data points.

6. **Given** no clinical documents have been uploaded for the patient, **When** the 360-degree view loads, **Then** the system displays an empty profile with a message "No clinical data available".

## Edge Cases

- What happens when extracted data has extremely low confidence across all documents? (All low-confidence fields are visually marked; staff sees "Unverified" badge on each field)
- How does the system handle a patient with 50+ uploaded documents? (Pagination or lazy loading of data sections; aggregation pipeline optimized for large document sets)
