# User Story - US_035

## Story ID

- ID: US_035

## Story Title

- Data Conflict Identification and Display

## Description

- As a Staff member, I want the system to explicitly identify and highlight critical data conflicts across patient documents with side-by-side comparison, so that I can review and understand discrepancies that may pose patient safety risks.

## Acceptance Criteria

1. **Given** multiple documents contain conflicting values for the same data field (e.g., different medication lists, differing allergy records), **When** the conflict detection runs, **Then** the system creates DataConflict records with both values and their source document references.

2. **Given** conflicts are detected, **When** I view the 360-degree patient profile, **Then** conflicting data points are visually highlighted with distinct conflict indicators (e.g., warning icons, colored borders).

3. **Given** I click on a conflict indicator, **When** the conflict detail opens, **Then** the system displays side-by-side values from different source documents with document name, page number, and extraction confidence.

4. **Given** retrieved chunks are used for conflict analysis, **When** the RAG pipeline processes them, **Then** semantic re-ranking is applied to retrieved chunks before AI prompt construction to maximize detection accuracy.

5. **Given** a critical conflict exists (e.g., conflicting medications that could interact), **When** the 360-degree view loads, **Then** the system displays a prominent conflict alert banner prompting immediate staff resolution.

## Edge Cases

- What happens when a conflict involves data from a low-confidence extraction? (Conflict is flagged with an additional "Low Confidence" indicator suggesting the conflicting value may be unreliable)
- How does the system handle three-way conflicts across three documents? (System creates pairwise DataConflict records and displays all three values in the comparison view)
