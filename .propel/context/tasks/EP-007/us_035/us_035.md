# User Story - US_035

## Story ID

- ID Format: US_035

## Story Title

- Google Calendar OAuth 2.0 Appointment Sync

## Description

- As a Patient, I want to sync my confirmed appointment to Google Calendar using OAuth 2.0 so that my appointment appears alongside my personal commitments automatically.

## Acceptance Criteria

1. **Given** I click "Add to Google Calendar" on my booking confirmation or dashboard, **When** I complete the OAuth 2.0 authorization flow, **Then** a calendar event is created in my primary Google Calendar with appointment date, start/end time, provider specialty, clinic name, appointment type, and booking reference number.

2. **Given** my Google Calendar event is created, **When** the event is confirmed by the Google Calendar API, **Then** a CalendarSync record is stored with `provider = Google`, `externalEventId`, `syncStatus = Synced`, and a link to the Google Calendar event is shown to me.

3. **Given** I decline OAuth authorization, **When** the authorization is denied, **Then** no calendar event is created, a guidance message is shown explaining how to grant permission, and the booking remains confirmed.

4. **Given** the Google Calendar API is temporarily unavailable, **When** my sync attempt fails, **Then** the CalendarSync record is stored with `syncStatus = Failed`, a retry is queued for 10 minutes, and an ICS download fallback is offered.

## Edge Cases

- What happens when my Google OAuth token expires? (Token refresh is attempted automatically; if refresh fails, the user is prompted to re-authorize)
- What happens if the same appointment is synced twice? (The existing CalendarSync record is detected; the second sync updates the existing Google Calendar event instead of creating a duplicate)
