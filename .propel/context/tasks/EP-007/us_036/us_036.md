# User Story - US_036

## Story ID

- ID: US_036

## Story Title

- Staff Data Verification Workflow

## Description

- As a Staff member, I want to confirm, reject, or modify AI-extracted data points within the 360-degree patient view, so that only staff-verified data is committed to the patient's consolidated record following the Trust-First approach.

## Acceptance Criteria

1. **Given** I am viewing the 360-degree patient profile with AI-extracted data, **When** I review a data point, **Then** I can accept (confirm), reject, or modify the value with the original AI-suggested value preserved for audit.

2. **Given** I accept an extracted data point, **When** the confirmation is saved, **Then** the system commits the verified value to the patient's consolidated record and logs the verification action in the audit trail.

3. **Given** I reject an extracted data point, **When** the rejection is saved, **Then** the system marks the data point as rejected, removes it from the consolidated view, and logs the rejection in the audit trail.

4. **Given** I modify an extracted data point, **When** I enter the corrected value and save, **Then** the system stores my corrected value as the verified value, preserves the original AI value for comparison, and logs the modification in the audit trail.

5. **Given** I resolve a data conflict, **When** I select the correct value or enter a corrected value and confirm, **Then** the DataConflict status is updated to "Resolved" with resolvedBy, resolvedAt, and resolvedValue recorded.

6. **Given** I cannot determine the correct value for a conflict, **When** I mark it as "Pending Review", **Then** the system updates the conflict status to "PendingReview" for follow-up.

## Edge Cases

- What happens when two staff members simultaneously verify the same data point? (Optimistic locking prevents conflicts; second verification receives a notification that the data was already verified)
- How does the system handle reverting a previously verified data point? (Staff can re-open verification for a data point; audit trail records the revert action)
