# User Story - US_037

## Story ID

- ID Format: US_037

## Story Title

- Calendar Event Update & Removal on Reschedule/Cancel

## Description

- As a Patient with a synced calendar, I want my calendar event to be automatically updated when I reschedule and deleted when I cancel my appointment, so that my calendar always reflects the current appointment state without manual effort.

## Acceptance Criteria

1. **Given** I reschedule a confirmed appointment that has an active CalendarSync record, **When** the reschedule is committed, **Then** the system calls the calendar API (Google PATCH `/events/{eventId}` or Outlook PATCH `/me/events/{id}`) with the new date and time, and the CalendarSync record is updated to `syncStatus = Synced`.

2. **Given** I cancel an appointment with an active CalendarSync record, **When** the cancellation is committed, **Then** the system calls the calendar API to delete the event (Google DELETE or Outlook DELETE), and the CalendarSync record is updated to `syncStatus = Revoked`.

3. **Given** the calendar API call fails during update or deletion, **When** the API returns an error, **Then** the CalendarSync record is updated to `syncStatus = Failed`, a retry is queued for 10 minutes, and the appointment status change proceeds regardless.

4. **Given** an appointment has no associated CalendarSync record, **When** I reschedule or cancel, **Then** no calendar API call is attempted and the cancellation/reschedule proceeds normally.

## Edge Cases

- What happens if the patient's calendar OAuth token has expired at the time of update propagation? (Token refresh is attempted; if refresh fails, the CalendarSync is marked `syncStatus = Failed` and the patient is prompted to reconnect)
- What happens if a batch of cancellations occurs (e.g., clinic closure) affecting many synced appointments? (Individual calendar API calls are queued and processed asynchronously; rate limits are respected per API provider)
