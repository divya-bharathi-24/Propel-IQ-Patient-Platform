# User Story - US_040

## Story ID

- ID Format: US_040

## Story Title

- AI Document Extraction RAG Pipeline

## Description

- As a system, I want each uploaded clinical PDF to be processed asynchronously through an AI extraction pipeline (chunking, embedding, vector storage, GPT-4o RAG extraction) so that structured clinical data (vitals, medications, diagnoses, allergies) is extracted with per-field confidence scores and source citations.

## Acceptance Criteria

1. **Given** a ClinicalDocument is stored with `processingStatus = Pending`, **When** the async extraction pipeline triggers, **Then** the document is chunked into 512-token segments with 10% overlap, embeddings are generated using `text-embedding-3-small`, and all chunks are stored in pgvector.

2. **Given** chunks are stored, **When** an extraction query runs, **Then** the top-5 chunks with cosine similarity ≥0.7 are retrieved, re-ranked by semantic relevance, and used to construct a GPT-4o prompt within the 8,000-token budget.

3. **Given** the GPT-4o extraction completes, **When** the structured JSON response is returned, **Then** ExtractedData records are created for each clinical field with: `value`, `confidence` (0–1), `sourcePageNumber`, `sourceTextSnippet`, and `documentId`; fields with confidence below 80% are flagged for priority staff review.

4. **Given** the extraction pipeline completes, **When** the final status is set, **Then** the ClinicalDocument record is updated to `processingStatus = Completed` and the patient is notified via email that their document has been processed.

## Edge Cases

- What happens when a PDF has no extractable text (scanned image-only PDF)? (OCR pre-processing is attempted; if OCR fails, `processingStatus = Failed` and a notification advises the patient to upload a text-based PDF)
- What happens when the OpenAI API circuit breaker is open? (Processing is paused; document status remains `Pending`; it is retried when the circuit breaker resets after the 5-minute window per AIR-O02)
