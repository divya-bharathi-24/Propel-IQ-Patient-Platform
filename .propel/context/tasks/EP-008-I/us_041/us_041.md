# User Story - US_041

## Story ID

- ID Format: US_041

## Story Title

- 360-Degree Patient View Aggregation & Staff Verification

## Description

- As an authenticated Staff member, I want to view an aggregated 360-degree patient view with all extracted clinical data de-duplicated, source-cited, and conflict-flagged — and verify it with a single explicit action — so that I can complete clinical preparation within 2 minutes for patients with up to 10 uploaded documents.

## Acceptance Criteria

1. **Given** I open the 360-degree view for a patient with processed documents, **When** the page loads within 2 minutes, **Then** I see aggregated sections for Vitals, Medications, Diagnoses, Allergies, Immunizations, and Surgical History with all duplicates collapsed into single entries.

2. **Given** a data element is extracted from a document, **When** I view it in the 360-degree view, **Then** each element shows a source citation (document name, page number, uploaded timestamp) and a confidence indicator; fields below 80% confidence are visually flagged for priority review.

3. **Given** I have reviewed all fields and resolved all conflicts (if any), **When** I click "Verify Profile", **Then** the profile status is updated to `Verified`, my staff ID and verification timestamp are recorded, and an audit event is logged.

4. **Given** the profile has unresolved Critical conflicts, **When** I attempt to click "Verify Profile", **Then** the action is blocked with a message listing each unresolved conflict that must be addressed first.

## Edge Cases

- What happens when a patient has more than 10 uploaded documents? (The 2-minute SLA target applies to ≤10 documents per FR-048; beyond 10, a progress indicator is shown and verification is still possible but not SLA-bound)
- What happens if extraction fails for one document in a set? (The 360-degree view shows data from successfully processed documents; the failed document is indicated with a "Processing Failed" badge and a retry option)
