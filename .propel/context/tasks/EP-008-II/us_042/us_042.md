# User Story - US_042

## Story ID

- ID Format: US_042

## Story Title

- ICD-10 & CPT Code AI Suggestion Engine

## Description

- As a system, I want the AI module to analyze a patient's aggregated clinical data and suggest applicable ICD-10 diagnostic codes and CPT procedure codes with per-code confidence scores and source document references, so that Staff can efficiently review and confirm codes for each encounter.

## Acceptance Criteria

1. **Given** a patient has a verified or processing 360-degree view, **When** the medical coding request is triggered, **Then** the Semantic Kernel tool-calling pipeline queries the ICD-10 code library and returns suggested codes with `code`, `description`, `confidence`, and `sourceDocumentId`.

2. **Given** the ICD-10 analysis completes, **When** the CPT procedure analysis runs, **Then** CPT code suggestions are returned with `codeType = CPT`, `confidence`, `description`, and mapped evidence from the patient's clinical documentation.

3. **Given** the AI generates code suggestions, **When** I inspect the output, **Then** all output conforms to the structured JSON schema (≥99% schema validity per AIR-Q03) and no hallucinated codes that are not part of the ICD-10 or CPT standard libraries are included.

4. **Given** confidence for a suggested code falls below 80%, **When** the code is displayed to Staff, **Then** it is visually flagged as "Low Confidence — Review Required" with the supporting evidence highlighted for scrutiny.

## Edge Cases

- What happens if the patient has no clinical documents yet? (Code suggestion endpoint returns an empty set with a message: "No clinical data available for code analysis — upload documents first")
- What happens if the ICD-10/CPT code library lookup fails? (Circuit breaker activates; tool calling is retried once; if it fails again, the Staff member is notified to retry and manual code entry is available)
