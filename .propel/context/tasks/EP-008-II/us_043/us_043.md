# User Story - US_043

## Story ID

- ID Format: US_043

## Story Title

- Medical Code Staff Review & Confirmation Interface

## Description

- As an authenticated Staff member, I want to review all AI-suggested ICD-10 and CPT codes in a side-by-side interface (code, description, supporting evidence) and explicitly confirm or reject each one, so that only clinically verified codes are stored for the patient encounter.

## Acceptance Criteria

1. **Given** I open the medical coding interface for a patient encounter, **When** the suggested codes load, **Then** I see a side-by-side view of ICD-10 codes on the left and CPT codes on the right, each showing the code, description, confidence score, and supporting evidence text from the source document.

2. **Given** I review an individual code, **When** I click "Confirm", **Then** the MedicalCode record is updated with `verificationStatus = Accepted`, `verifiedBy = staffId`, and `verifiedAt = UTC timestamp`.

3. **Given** I review an individual code and click "Reject", **When** the rejection is submitted, **Then** the MedicalCode record is updated with `verificationStatus = Rejected` and a rejection reason is logged with my staff ID.

4. **Given** I need to add a code that was not suggested by AI, **When** I enter a manual ICD-10 or CPT code, **Then** the system validates the code format against the standard code library, and upon validation, creates a new MedicalCode record with `verificationStatus = Accepted` and a manual entry flag.

## Edge Cases

- What happens if I try to submit the medical coding review with unreviewed codes remaining? (System allows partial submission and tracks which codes are still `Pending`; a progress indicator shows "X codes pending review")
- What happens when the same encounter has codes reviewed by multiple Staff members? (The most recent reviewer's decision is recorded; all previous review actions are preserved in the audit log)
