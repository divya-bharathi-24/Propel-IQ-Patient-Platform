# User Story - US_037

## Story ID

- ID: US_037

## Story Title

- ICD-10 Diagnosis Code Suggestion

## Description

- As a clinical staff member, I want the system to extract and suggest applicable ICD-10 diagnosis codes from aggregated patient data and clinical documents with confidence scores, so that I can review AI-suggested codes instead of manually identifying diagnosis codes.

## Acceptance Criteria

1. **Given** a patient has processed clinical documents with extracted data, **When** the AI coding engine runs, **Then** the system suggests applicable ICD-10 diagnosis codes with per-code confidence scores.

2. **Given** ICD-10 codes are suggested, **When** the results are displayed, **Then** each code includes the ICD-10 code identifier, description, confidence score, and reference to the source document.

3. **Given** the AI generates code suggestions, **When** extraction completes, **Then** MedicalCode records are created with codeType "ICD10", confidence scores, source document reference, and verificationStatus "Pending".

4. **Given** the AI coding engine processes patient data, **When** codes are generated, **Then** the processing completes within 30 seconds (p95 latency).

5. **Given** insufficient patient data exists for code generation, **When** the coding engine runs, **Then** the system returns an empty suggestion list with a message "Insufficient data for ICD-10 code suggestions".

## Edge Cases

- What happens when the AI suggests a deprecated ICD-10 code? (Code validation checks against current ICD-10 code set; deprecated codes are flagged with replacement suggestions)
- How does the system handle conflicting diagnoses across documents? (All applicable codes are suggested with source references; staff resolves conflicts during review)
