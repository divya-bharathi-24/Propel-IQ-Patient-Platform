# User Story - US_038

## Story ID

- ID: US_038

## Story Title

- CPT Procedure Code Mapping

## Description

- As a clinical staff member, I want the system to map and suggest applicable CPT procedure codes from clinical documentation with confidence scores, so that I can review AI-suggested procedure codes instead of manually identifying them from clinical notes.

## Acceptance Criteria

1. **Given** a patient has processed clinical documents, **When** the AI coding engine runs, **Then** the system maps and suggests applicable CPT procedure codes with per-code confidence scores.

2. **Given** CPT codes are suggested, **When** the results are displayed, **Then** each code includes the CPT code identifier, description, confidence score, and reference to the source document.

3. **Given** the AI generates CPT suggestions, **When** extraction completes, **Then** MedicalCode records are created with codeType "CPT", confidence scores, source document reference, and verificationStatus "Pending".

4. **Given** the AI coding engine processes patient data, **When** codes are generated, **Then** the processing completes within 30 seconds (p95 latency).

5. **Given** insufficient clinical documentation exists, **When** the coding engine runs, **Then** the system returns an empty suggestion list with a message "Insufficient data for CPT code suggestions".

## Edge Cases

- What happens when a procedure is documented ambiguously? (AI suggests multiple candidate CPT codes with lower confidence; staff selects the correct one during review)
- How does the system handle procedures not in the standard CPT code set? (System flags as "Unlisted Procedure" with closest matching codes suggested)
