# User Story - US_041

## Story ID

- ID: US_041

## Story Title

- Risk Score Dashboard Display

## Description

- As a Staff member, I want to see the no-show risk score displayed on the scheduling dashboard for each appointment, so that I can make informed scheduling decisions and target high-risk patients with additional reminders or outreach.

## Acceptance Criteria

1. **Given** I am authenticated as Staff, **When** I view the scheduling dashboard, **Then** each appointment displays its calculated no-show risk score with a visual indicator (e.g., green/yellow/red for low/medium/high risk).

2. **Given** risk scores are displayed, **When** I sort or filter the dashboard, **Then** I can filter appointments by risk level (High, Medium, Low) to focus on high-risk patients.

3. **Given** a high-risk appointment is identified, **When** I view its risk details, **Then** the contributing factors (appointment history, lead time, reminder engagement) are displayed to inform my intervention strategy.

4. **Given** the dashboard loads, **When** risk scores exist for today's appointments, **Then** the scores are displayed within the standard 2-second API response time.

5. **Given** no risk score has been calculated for an appointment, **When** the dashboard loads, **Then** the appointment displays "Risk: Not Calculated" instead of an empty value.

## Edge Cases

- What happens when risk scores are recalculated while the dashboard is open? (Dashboard refreshes on next data fetch; stale scores are replaced with updated values)
- How does the system handle a dashboard with hundreds of appointments? (Pagination with configurable page size; default shows today's appointments)
