# User Story - US_045

## Story ID

- ID Format: US_045

## Story Title

- Admin CRUD on Staff & Admin User Accounts

## Description

- As an authenticated Admin, I want to create, view, edit, and soft-delete Staff and Admin user accounts, with a credential setup email sent on creation, so that I have full lifecycle control over the team's access to the platform.

## Acceptance Criteria

1. **Given** I am authenticated as Admin and navigate to User Management, **When** the page loads, **Then** I see a list of all Staff and Admin accounts with name, email, role, status (Active/Deactivated), and last login date.

2. **Given** I create a new Staff account with name, email, and role, **When** the account is saved, **Then** the User record is created with `status = Active`, a secure credential setup email is sent to the new user, and the action is logged in the audit trail with before/after state.

3. **Given** I choose to deactivate an account, **When** the deactivation is confirmed (after re-authentication per US_046), **Then** the User record is updated with `status = Deactivated` (soft-delete; row is retained), the user's active sessions are invalidated immediately, and the action is audited.

4. **Given** a non-Admin role user attempts to call user management endpoints, **When** the request is evaluated, **Then** HTTP 403 Forbidden is returned and no data is modified.

## Edge Cases

- What happens if I try to deactivate my own Admin account? (System prevents self-deactivation with a validation error: "Cannot deactivate your own account")
- What happens when the credential setup email fails to send? (Account is still created; Admin sees a "Email delivery failed" warning and can resend the credential email from the user detail page)
