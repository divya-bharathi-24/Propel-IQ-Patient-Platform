# User Story - US_047

## Story ID

- ID Format: US_047

## Story Title

- Read-Only Immutable Audit Log Interface

## Description

- As an authenticated Admin, I want to view a read-only audit log capturing all patient data access events, clinical modifications, authentication events, and user management actions, so that I can perform HIPAA-compliant access reviews and investigate any suspicious activity.

## Acceptance Criteria

1. **Given** I navigate to the Audit Log interface as Admin, **When** the log loads, **Then** I see a paginated, time-ordered list of audit events with: user ID, role, entity type, entity ID, action type (Create/Read/Update/Delete), IP address, and UTC timestamp.

2. **Given** audit log events are displayed, **When** I apply filters (by date range, user ID, action type, or entity type), **Then** only matching events are shown in the filtered view without page reload.

3. **Given** I view an audit log event for a clinical data modification, **When** I expand the event, **Then** the before-and-after state of the modified entity is shown in a structured diff view.

4. **Given** any user role other than Admin attempts to access the audit log endpoint, **When** the request is evaluated, **Then** HTTP 403 Forbidden is returned; no audit data is exposed.

## Edge Cases

- What happens when the audit log has millions of records and pagination is needed? (Cursor-based pagination is used for performance; page sizes are limited to 50 records per page with a total count displayed)
- What happens if I try to export or download the audit log from within the application? (Export functionality is intentionally excluded per FR-059; the UI contains no download or export button)
