# User Story - US_048

## Story ID

- ID Format: US_048

## Story Title

- AI Quality Monitoring & Hallucination Rate Control

## Description

- As a system operator, I want the platform to track the AI-Human Agreement Rate (target ≥98%), enforce output schema validity (≥99%), monitor hallucination rates (<2%), and enforce per-field confidence thresholds (80% floor), so that AI outputs meet clinical safety and quality standards.

## Acceptance Criteria

1. **Given** staff confirm or reject AI-suggested codes and clinical data fields, **When** the decision is logged, **Then** an AI agreement metric event is recorded for the associated AI extraction session so the overall AI-Human Agreement Rate is calculable.

2. **Given** the AI generates a structured JSON output, **When** the schema validation middleware runs, **Then** outputs failing schema validation are rejected (not persisted), an alert is logged, and the request falls back to manual review — maintaining ≥99% schema validity.

3. **Given** staff verify AI-extracted data against ground truth, **When** discrepancies are recorded, **Then** the hallucination rate metric is updated; if the rolling rate exceeds 2%, a critical alert is raised for model review.

4. **Given** any extracted field has confidence below 80%, **When** the field is evaluated by the confidence gate, **Then** the field is flagged for priority staff review and is not auto-committed to the patient record.

## Edge Cases

- What happens when there is not enough ground truth data to calculate the hallucination rate? (Rate is displayed as "Insufficient data" until a minimum of 50 verified samples are accumulated)
- What happens when the AI-Human Agreement Rate drops below 98%? (Alert is raised to the admin; a flag is set that surfaces on the AI operational dashboard; no automatic model rollback occurs in Phase 1)
