# User Story - US_050

## Story ID

- ID Format: US_050

## Story Title

- AI Operational Controls — Circuit Breaker, Token Budget & Model Swap

## Description

- As a system operator, I want a circuit breaker that falls back to manual workflow after 3 consecutive AI failures in 5 minutes, an 8,000-token per request budget enforced, and the ability to change the AI model version without redeployment (effective within 5 minutes), so that the platform is resilient to AI provider outages and operationally efficient.

## Acceptance Criteria

1. **Given** the AI provider returns 3 consecutive failures within a 5-minute window, **When** the circuit breaker trips, **Then** subsequent AI requests are short-circuited for 5 minutes, a manual fallback prompt is displayed to staff, and a circuit breaker alert is logged.

2. **Given** an AI request is constructed, **When** the token counter evaluates the prompt, **Then** prompts exceeding 8,000 tokens are truncated (lower-priority context removed first) and the truncation event is logged; no request exceeding the budget is sent to the provider.

3. **Given** an operator changes the AI model version in the system configuration, **When** the configuration is saved, **Then** the new model version is used for all subsequent AI requests within 5 minutes — without requiring an application restart or redeployment.

4. **Given** AI operational metrics are being collected, **When** I view the AI metrics dashboard, **Then** I see token consumption per request, p95 latency (target ≤30s), error rates, circuit breaker trips, and confidence score distributions — all updated within 60 seconds.

## Edge Cases

- What happens if the circuit breaker resets but the AI provider is still unstable? (Three more failures within 5 minutes re-trip the circuit breaker; exponential backoff is applied for repeated trips within the same hour)
- What happens if token truncation removes clinically relevant context? (Truncation removes the oldest/lowest-similarity chunks first per the RAG re-ranking order; a warning is logged indicating which chunks were dropped)
