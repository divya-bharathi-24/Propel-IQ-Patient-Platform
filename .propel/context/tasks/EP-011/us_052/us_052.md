# User Story - US_052

## Story ID

- ID Format: US_052

## Story Title

- Platform Availability & Graceful Degradation Handlers

## Description

- As a platform engineer, I want the system to maintain 99.9% monthly uptime with health check endpoints and graceful degradation handlers for all external service dependencies (calendar APIs, SMS, email, AI providers), so that core booking and clinical workflows remain functional even when external services are unavailable.

## Acceptance Criteria

1. **Given** the platform is running, **When** I call `GET /health`, **Then** the endpoint returns HTTP 200 with a structured health report listing the status of each dependency (PostgreSQL, Redis, SendGrid, Twilio, OpenAI, Google Calendar, Microsoft Graph).

2. **Given** SendGrid is unavailable, **When** an email notification is attempted, **Then** the notification is queued for retry (up to 3 attempts with exponential backoff), the booking confirmation or reminder is not blocked, and the user sees a "Notification pending delivery" status.

3. **Given** the OpenAI API is unavailable, **When** an AI extraction or coding request is made, **Then** the circuit breaker activates (per US_050), core booking and document upload continue to function, and staff are shown a "AI processing temporarily unavailable — manual review required" message.

4. **Given** a Google or Outlook Calendar API is unavailable, **When** a calendar sync is attempted, **Then** the CalendarSync record is set to `syncStatus = Failed`, the appointment booking is confirmed without blocking, and the ICS download fallback is offered to the user.

## Edge Cases

- What happens during a planned maintenance window? (Maintenance is communicated via a system banner; booking and intake remain available; AI and calendar sync features are temporarily suspended with user-facing notices)
- What happens if PostgreSQL is unreachable? (Health endpoint returns HTTP 503; all API endpoints that require DB access return 503 with a structured error; a critical alert fires immediately)
