# User Story - US_007

## Story ID

- ID: US_007

## Story Title

- Appointment and Scheduling Entities

## Description

- As a developer, I want the Appointment, WaitlistEntry, Specialty, QueueEntry, and Notification entities implemented with EF Core relationships and status enums, so that the scheduling domain model supports booking, waitlist, queue, and notification workflows with data integrity.

## Acceptance Criteria

1. **Given** the Appointment entity is defined, **When** it is configured, **Then** it includes all attributes (id, patientId, specialtyId, date, timeSlotStart, timeSlotEnd, status, cancellationReason, createdBy, createdAt) with status restricted to Booked/Arrived/Cancelled/Completed and FK relationships to Patient and Specialty.

2. **Given** the WaitlistEntry entity is defined, **When** it is configured, **Then** it includes FIFO ordering enforced via enrolledAt timestamp index with status restricted to Active/Swapped/Expired and FK relationships to Patient and Appointment.

3. **Given** the Specialty entity is defined, **When** it is configured, **Then** it includes id, name, and description attributes with a one-to-many relationship to Appointment.

4. **Given** the QueueEntry entity is defined, **When** it is configured, **Then** it includes position (integer), arrivalTime, and status (Waiting/Called/Removed) with a one-to-one relationship to Appointment and many-to-one to Patient.

5. **Given** all scheduling entities are configured, **When** a migration is generated and applied, **Then** all tables are created with correct FK constraints, indexes on frequently queried columns (patientId, date, status), and the Appointment soft deletion filter.

## Edge Cases

- What happens when deleting a Specialty that has existing Appointments? (FK constraint prevents deletion; application requires reassignment or deactivation)
- How does the system handle WaitlistEntry referencing a cancelled Appointment? (Status transition logic updates WaitlistEntry to Expired when linked Appointment is cancelled)
