# User Story - US_009

## Story ID

- ID: US_009

## Story Title

- Clinical and AI Data Entities

## Description

- As a developer, I want the ClinicalDocument, ExtractedData, DataConflict, MedicalCode, AuditLog, and NoShowRisk entities implemented with confidence scoring, immutable audit constraints, and relationship mappings, so that clinical intelligence and compliance data models are ready for AI extraction and audit workflows.

## Acceptance Criteria

1. **Given** the ClinicalDocument entity is defined, **When** it is configured, **Then** it includes encrypted storagePath, processingStatus (Pending/Processing/Completed/Failed), and FK to Patient with a one-to-many relationship to ExtractedData.

2. **Given** the ExtractedData entity is defined, **When** it is configured, **Then** it includes dataType enum (Vital/History/Medication/Allergy/Diagnosis), confidence decimal (0-1), sourcePageNumber, sourceTextSnippet, and FK relationships to ClinicalDocument and Patient.

3. **Given** the DataConflict entity is defined, **When** it is configured, **Then** it includes side-by-side values (value1/value2) with source document FKs, resolutionStatus (Unresolved/Resolved/PendingReview), and resolution tracking fields (resolvedValue, resolvedBy, resolvedAt).

4. **Given** the MedicalCode entity is defined, **When** it is configured, **Then** it includes codeType (ICD10/CPT), confidence decimal, verificationStatus (Pending/Accepted/Rejected/Modified), and FK relationships to Patient and source ClinicalDocument.

5. **Given** the AuditLog entity is defined, **When** it is configured, **Then** the table is configured as INSERT-only at the database level (no UPDATE or DELETE triggers), includes JSONB details field, and has FK to User.

6. **Given** the NoShowRisk entity is defined, **When** it is configured, **Then** it includes score decimal (0-1), JSONB factors field, and a one-to-one relationship to Appointment.

## Edge Cases

- What happens when an UPDATE or DELETE is attempted on the AuditLog table? (Database-level trigger or policy rejects the operation; application uses a write-only repository pattern)
- How does the system handle ExtractedData with a confidence of exactly 0.80? (Boundary condition: confidence >= 0.80 passes auto-accept threshold; < 0.80 requires manual review)
