# User Story - US_001

## Story ID

- ID Format: US_001

## Story Title

- Angular 18 SPA Workspace Setup

## Description

- As a developer, I want an Angular 18 SPA workspace configured with standalone components, signals, and NgRx Signals, so that the team can begin building patient-facing and staff-facing frontend features immediately.

## Acceptance Criteria

1. **Given** I clone the repository, **When** I run `npm install && ng serve`, **Then** the Angular 18 dev server starts without errors and displays a bootstrapped shell application.

2. **Given** the Angular workspace is initialized, **When** I inspect the project structure, **Then** standalone components are the default, Angular Signals are enabled, and NgRx Signals store is scaffolded with at least one feature slice.

3. **Given** the workspace is set up, **When** I run `ng build --configuration production`, **Then** the production build completes with LCP-optimized bundle output and no TypeScript compile errors.

4. **Given** the workspace is configured, **When** I run `ng lint`, **Then** ESLint rules pass with zero errors across all scaffold files.

## Edge Cases

- What happens when a developer uses an older Node.js version incompatible with Angular 18? (Document minimum Node.js version in README; fail fast with an engine constraint in package.json)
- How does the system handle multiple developers running the dev server simultaneously? (Ports are configurable via angular.json; default port 4200 documented)
