# User Story - US_001

## Story ID

- ID: US_001

## Story Title

- Angular 18 Frontend Workspace Setup

## Description

- As a developer, I want a properly scaffolded Angular 18 workspace with standalone components, signals, and NgRx Signals state management, so that frontend development can begin with modern Angular patterns and consistent project structure.

## Acceptance Criteria

1. **Given** the project repository is cloned, **When** the Angular workspace is initialized, **Then** the workspace uses Angular 18.x with standalone components enabled by default and strict TypeScript mode configured.

2. **Given** the Angular workspace is created, **When** the project structure is reviewed, **Then** it contains separate feature module directories for Patient, Staff, and Admin dashboards with lazy-loaded routing.

3. **Given** the Angular workspace is created, **When** NgRx Signals is installed and configured, **Then** a root signal store is initialized with a sample state slice demonstrating the signals pattern.

4. **Given** the development server is started, **When** a developer navigates to localhost, **Then** the application renders a placeholder landing page within 3 seconds (LCP < 2.5s).

5. **Given** the Angular workspace is created, **When** the linting and formatting tools are reviewed, **Then** ESLint and Prettier are configured with rules enforcing Angular coding standards and strict TypeScript.

## Edge Cases

- What happens when Node.js version is incompatible with Angular 18? (Document required Node.js version in README; fail build with clear error message)
- How does the system handle missing npm dependencies during fresh install? (Lock file ensures deterministic installs; CI validates with clean install)
