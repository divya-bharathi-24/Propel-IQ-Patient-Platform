# User Story - US_003

## Story ID

- ID: US_003

## Story Title

- PostgreSQL Database and EF Core Setup

## Description

- As a developer, I want PostgreSQL 16+ provisioned with Entity Framework Core 8 configured for code-first migrations, so that the data persistence layer is ready for domain entity implementation with reliable schema management.

## Acceptance Criteria

1. **Given** the development environment is set up, **When** PostgreSQL is provisioned, **Then** a PostgreSQL 16+ instance is accessible via connection string with TLS enabled.

2. **Given** PostgreSQL is running, **When** Entity Framework Core 8 is configured, **Then** the DbContext is registered in the DI container with Npgsql provider and connection pooling enabled.

3. **Given** EF Core is configured, **When** a developer runs `dotnet ef migrations add InitialCreate`, **Then** a migration is generated successfully and can be applied to the database without errors.

4. **Given** the migration pipeline is set up, **When** `dotnet ef database update` is executed, **Then** the database schema is created and the migration history table records the applied migration.

5. **Given** the database connection is configured, **When** the API service starts, **Then** the health check endpoint at `/health` verifies database connectivity and returns a healthy status.

## Edge Cases

- What happens when PostgreSQL is unreachable during application startup? (Health check reports unhealthy; application logs connection failure with retry guidance)
- How does the system handle concurrent migration attempts? (EF Core migration lock prevents parallel execution; second attempt waits or fails gracefully)
