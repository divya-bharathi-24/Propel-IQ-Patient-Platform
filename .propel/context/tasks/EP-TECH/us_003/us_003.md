# User Story - US_003

## Story ID

- ID Format: US_003

## Story Title

- Docker Compose Local Development Environment

## Description

- As a developer, I want a Docker Compose configuration that orchestrates the frontend, backend, PostgreSQL, and Redis locally, so that I can run the full platform stack with a single command and maintain consistent local environments across the team.

## Acceptance Criteria

1. **Given** Docker Desktop is running, **When** I execute `docker compose up`, **Then** all four services (Angular dev server, .net 10 API, PostgreSQL 16 with pgvector, Upstash Redis emulator) start within 2 minutes without manual intervention.

2. **Given** the Docker Compose environment is running, **When** the backend initializes, **Then** it applies pending EF Core migrations automatically and seeds the Specialty reference table.

3. **Given** the local stack is up, **When** I navigate to the Swagger UI URL, **Then** the API gateway is reachable and all health check endpoints return HTTP 200.

4. **Given** the stack is running, **When** I stop a single service (e.g., Redis), **Then** the remaining services continue running and the API returns a graceful degradation response for Redis-dependent endpoints.

## Edge Cases

- What happens when the PostgreSQL container starts before migrations are ready? (Health check with `depends_on` and retry logic ensures migrations run after DB is ready)
- How are environment variables (secrets) handled locally? (`.env.example` file provided; `.env` excluded from git via `.gitignore`)
