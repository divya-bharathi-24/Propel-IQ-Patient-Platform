# Migration Conflict Fixed - Pending Migration Removed

## ? Root Cause Found and Fixed

### The Problem
The error was caused by a **pending migration** (`AddPatientPendingAlerts_v2`) that was trying to:
1. Drop the HNSW index `ix_document_chunk_embeddings_embedding_hnsw`
2. Alter the `embedding` column in `document_chunk_embeddings` table

But since we've disabled the `document_chunk_embeddings` table (it was never created), the migration was trying to drop an index that doesn't exist!

### Error Details
```
Npgsql.PostgresException: 42704: index "ix_document_chunk_embeddings_embedding_hnsw" does not exist
```

This error occurred at:
```csharp
Program.cs:line 1242
await db.Database.MigrateAsync();
```

## ?? Solution Applied

### Files Removed
1. ? `Propel.Api.Gateway\Migrations\20260428090900_AddPatientPendingAlerts_v2.cs`
2. ? `Propel.Api.Gateway\Migrations\20260428090900_AddPatientPendingAlerts_v2.Designer.cs`

### Why This Migration Had to Be Removed

The migration was auto-generated by EF Core and contained operations on the `document_chunk_embeddings` table:

```csharp
// This fails because the index doesn't exist!
migrationBuilder.DropIndex(
    name: "ix_document_chunk_embeddings_embedding_hnsw",
    table: "document_chunk_embeddings");

// This fails because the table doesn't exist!
migrationBuilder.AlterColumn<float[]>(
    name: "embedding",
    table: "document_chunk_embeddings",
    type: "real[]",
    nullable: false,
    oldClrType: typeof(Vector),
    oldType: "vector(1536)");
```

## ?? When Enabling pgvector Later

When you uncomment and enable pgvector:

1. **Enable pgvector extension**
   ```powershell
   .\setup-pgvector.ps1
   ```

2. **Uncomment all vector code**
   - `Program.cs` - all commented registrations
   - `AppDbContext.cs` - DbSet and extension
   - `DocumentChunkEmbeddingConfiguration.cs` - vector column config
   - `DocumentChunkEmbeddingRepository.cs` - change `#if false` to `#if true` or remove preprocessor
   - Migration file - uncomment table creation

3. **Create new migration if needed**
   ```powershell
   cd Propel.Api.Gateway
   dotnet ef migrations add EnablePgvectorSupport
   dotnet ef database update
   ```

4. **Restart**
   ```powershell
   .\restart-all.ps1
   ```
